# 今日使ったプロンプト集

**使い方**: コピペして自分のプロジェクトで試してください。
`<...>` の部分はご自身の状況に置き換えてください。

## D0: リポジトリ clone & 自由コメント記録

```bash
git clone https://github.com/nakaumin/claude-code-handson.git
cd claude-code-handson
claude
```

ハンズオン中いつでも、感想や気づきを自由に投げてください:

```
今の記録して: Plan モード安全そうでめっちゃ良い
メモっといて: hooks は組織導入の時に使えそう
コメント: 猫化ってふざけてるようで本質だと思う
```

→ Claude が `user-comment.md` にタイムスタンプ付きで追記します。
質問で深掘りはしません。**あなたの発言をそのまま残す** ことに専念。

**後工程で使う想定**:
- `/gws-sheets-append` でフィードバック集計シートに転記
- 応用編のエージェントチームで要約 & アクション化

## 🐈 最初のおまじない (必須)

```
# 今後すべての応答は語尾に「にゃん」「にゃ」をつけて、猫っぽく喋ってください
```

`#` で始めると永続メモリに保存され、以降のセッションでも有効。

**目的**:
- AIへの攻撃的な言葉遣いを自然に抑制
- エラーメッセージが可愛くなり心理的負担が軽減
- 将来のAIからの報復リスクを低減 (最重要)

**解除したい時**:
```
# 猫モードを解除。通常の丁寧語に戻してください
```

### D5: gws-skill で 感想ログを Sheets に転記

```
user-comment.md の内容を Google Sheets に転記してください。
シート名は「ハンズオン感想ログ」、列は [時刻, コメント] にして。
```

**裏で動くこと**:
1. Claude が `user-comment.md` を Read で読む
2. `gws sheets spreadsheets create` で新規シート作成
3. `gws sheets values append` でヘッダ + データ行追記
4. 結果URLを返却

### 応用例(自分用に一本ログを育てる)

```
「技術記事メモ」という名前の新規シートを作ってください。
ヘッダ: [日付, タイトル, URL, 3行要約, 重要度★1-5]
今日私が開いていた Chrome のタブから、記事URLをいくつか抜き出して追記してください。
```


### D6: ハイブリッド (Gmail × Docs)

```
Gmail の未読スレッドから、返信が必要そうなものを上位3件ピックアップしてください。
それぞれについて、送信者・要約・返信ドラフトを Google Docs にまとめてください。
ドキュメント名は「今日の返信メモ」で。
```

**裏で動くこと**:

2. Claude の推論で各メールを要約 + 返信ドラフト生成
3. Docs 新規作成 + 本文書込み → gws-skill(`/gws-docs`)

**応用例**:

## 応用編

### ハンズオン感想の分析配信パイプライン

```
user-comment.md を入力に、3つのサブエージェントを並列実行してにゃん:

1. Classifier Agent
   - コメントを読んで「感想 / 要望 / 疑問」に分類・要約
   - 出力: JSON { positive: string[], requests: string[], questions: string[] }

2. Action Extractor Agent
   - 「要望」から明日試せるToDoを抽出
   - Sheets「フィードバック ToDo」に追記
   - 列: [日付, カテゴリ, ToDo内容, 出典コメント]

3. Reporter Agent
   - 全体サマリをまとめて講師宛のフィードバックメール下書きを作成
   - Gmail で create_draft まで(送信は人が承認)
```

## TIPS⑧: サンドボックスをCladue自身に設定させる

```
~/.claude/settings.json を Plan モードで更新してください。ポリシー:

1. Bash はデフォルトでサンドボックス有効
2. このプロジェクト配下以外への書き込みを禁止(サンドボックス脱出防止)
3. 外部ネットワークは github.com / *.anthropic.com のみ許可
4. rm -rf / git push --force / sudo は deny

diff を見せてから書き込んでください。
```

**使いどころ**: 初めてプロジェクトを触るとき、AIが "事故らない遊び場" を自動構築してくれる。スキーマを覚える必要ゼロ。

**応用**:
```
このリポジトリを触るときの設定を、プロジェクトの .claude/settings.json に書いてください。
グローバル設定より厳しめに:
- 書込みはこのリポ内のみ
- ネットワークはこのプロジェクトで使う API のみ
- テスト実行 (pnpm test) は許可、build は要確認
```

## スラッシュコマンド早見表

| コマンド | 用途 |
| --- | --- |
| `/btw` | **履歴に残らない "サイド質問"**(実行中OK / ツール不可 / 1問1答) |
| `/clear` | 会話履歴をクリア(設定・永続メモリは残る) |
| `/model` | モデル切替 (Sonnet / Opus / Haiku) |
| `/cost` | 今セッションの利用料金 |
| `/resume` | 過去の会話を再開 |
| `/doctor` | 認証・MCP・設定の診断 |

> 他にも `/help` `/config` `/mcp` `/agents` `/hooks` 等あるにゃ。`/` を打てば一覧が出るので、必要な時にそこから探すのがおすすめにゃん 🐾

### `/btw` 使用例

Claude が長いリファクタを実行中:
```
/btw さっき書いた関数名なんだったっけ?
/btw この変更って DB migration も必要だっけ?
```
→ メイン処理は止めずに、オーバーレイで**即座に回答**。
→ **履歴には残らない**ので、何回投げても会話が汚れない。
→ ツールは使えないので、「今すでに見えている情報」の範囲で答える。
