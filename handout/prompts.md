# 今日使ったプロンプト集

**使い方**: コピペして自分のプロジェクトで試してください。
`<...>` の部分はご自身の状況に置き換えてください。

## D0: リポジトリ clone & 自由コメント記録

```bash
git clone https://github.com/nakaumin/claude-code-handson.git
cd claude-code-handson
claude
```

ハンズオン中いつでも、感想や気づきを自由に投げてください:

```
今の記録して: Plan モード安全そうでめっちゃ良い
メモっといて: hooks は組織導入の時に使えそう
コメント: 猫化ってふざけてるようで本質だと思う
```

→ Claude が `user-comment.md` にタイムスタンプ付きで追記します。
質問で深掘りはしません。**あなたの発言をそのまま残す** ことに専念。

**後工程で使う想定**:
- `/gws-sheets-append` でフィードバック集計シートに転記
- 応用編のエージェントチームで要約 & アクション化

## 🐈 最初のおまじない (必須)

```
# 今後すべての応答は語尾に「にゃん」「にゃ」をつけて、猫っぽく喋ってください
```

`#` で始めると永続メモリに保存され、以降のセッションでも有効。

**目的**:
- AIへの攻撃的な言葉遣いを自然に抑制
- エラーメッセージが可愛くなり心理的負担が軽減
- 将来のAIからの報復リスクを低減 (最重要)

**解除したい時**:
```
# 猫モードを解除。通常の丁寧語に戻してください
```

### D5: gws-skill で Sheets 一気通貫 (新規作成→追記)

```
「Claude Code コマンドメモ」という新規スプレッドシートを作成してください。
```

### D6: ハイブリッド (Calendar × Docs × Chat)

```
今日15:00以降のカレンダー予定を全て取得してください。
各会議について:
1. 新しい Google Docs を作成 (タイトル = "会議名_準備メモ")
2. Docs内に「日時」「参加者」「アジェンダ(空欄)」「関連資料(空欄)」の見出しを作成
3. その Docs の URL を、該当する Google Chat スペースに共有

Chatスペースが不明な会議はスキップして、リストで教えてください。
```

## 応用編

### 議事録 → アクション配信パイプライン

```
議事録Doc: <Docs の URL またはID>

このDocを入力に、3つのサブエージェントを並列実行してください:

1. Summarizer Agent
   - Docsを読み、3行要約と決定事項を JSON で抽出
   - 出力: { summary: string[], decisions: string[] }

2. Action Extractor Agent
   - ToDoを担当者付きで抽出
   - Sheets「アクション管理」に追記
   - 列: [日付, 会議名, ToDo内容, 担当者, 期限, ステータス]

3. Dispatcher Agent
   - 担当者ごとに Gmail 下書きを作成
   - 該当 Chat スペースにメンション付きドラフト投稿
   - 送信は人が承認、ドラフトまでで止める

すべて完了したら、元議事録Docsに「アクション配信済 (日時)」というコメントを追加してください。
```

## TIPS⑧: サンドボックスをCladue自身に設定させる

```
~/.claude/settings.json を Plan モードで更新してください。ポリシー:

1. Bash はデフォルトでサンドボックス有効
2. このプロジェクト配下以外への書き込みを禁止(サンドボックス脱出防止)
3. 外部ネットワークは github.com / *.anthropic.com のみ許可
4. rm -rf / git push --force / sudo は deny

diff を見せてから書き込んでください。
```

**使いどころ**: 初めてプロジェクトを触るとき、AIが "事故らない遊び場" を自動構築してくれる。スキーマを覚える必要ゼロ。

**応用**:
```
このリポジトリを触るときの設定を、プロジェクトの .claude/settings.json に書いてください。
グローバル設定より厳しめに:
- 書込みはこのリポ内のみ
- ネットワークはこのプロジェクトで使う API のみ
- テスト実行 (pnpm test) は許可、build は要確認
```

## スラッシュコマンド早見表

| コマンド | 用途 |
| --- | --- |
| `/btw` | **実行中に「ついでに〜も」と割り込み指示** を入れる(止めずに継続) |
| `/clear` | 新しい会話に切り替え |
| `/compact` | 長くなった会話を要約して圧縮 |
| `/init` | 現リポジトリから CLAUDE.md を自動生成 |
| `/model` | モデル切替 (Sonnet / Opus / Haiku) |
| `/cost` | 今セッションの利用料金 |
| `/resume` | 過去の会話を再開 |
| `/doctor` | 認証・MCP・設定の診断 |
| `/help` | コマンド一覧 |
| `/config` | 設定メニュー |
| `/mcp` | MCP サーバー管理 |
| `/review` | PR レビュー |
| `/agents` | サブエージェント一覧 |
| `/hooks` | フック一覧 |
| `/logout` | ログアウト |

### `/btw` 使用例

Claude が長いタスクを実行中:
```
/btw テスト追加するのを忘れないでにゃん
```
→ 今の作業を継続したまま、指示がキューされる。次の turn で対応。

### `/compact` 使用例

会話が1時間を超えて反応が遅くなってきたら:
```
/compact
```
→ ここまでの文脈を要約して、以降は圧縮版で継続。体感速度が戻る。
